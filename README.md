# 簡易仕様書

# NearbyRestaurantBrochure
### リクルートWEBサービスのホットペッパーグルメリサーチAPIを使用して、現在地付近のレストラン情報を検索するスマートフォンアプリ

### 対象OS
iOS 26~

### 開発環境/言語
- 開発環境
  - Xcode 26.2
- 開発言語
  - Swift 6.2.3

### 開発期間
14日間

### 機能概要・一覧
- 検索条件入力画面
- 店舗一覧表示画面
- 店舗詳細画面
- 店舗の位置を地図で確認する機能

### 画面概要
- 店舗一覧画面
- 検索条件設定画面
- 店舗詳細画面

### 使用しているライブラリ、SDKなど
- SwiftUI
- Foundation
- CoreLocation
- Combine
- shimmer
- MapKit

### コンセプト
片手で全ての操作が行えることをメインテーマとし、パンフレットくらい気軽にお店を探せるアプリケーションを目指しました。

### こだわったポイント
AIのコード生成は参考までにして実装には自分で書いたもののみを使用しました。
また、長期的な運用を考えて基本的にネイティブのものを使用して開発を進めました。
issueで機能やデータなどの情報を管理しました。
勉強の意味も込めてなるべく責務を分離できるような設計をしました。
APIkeyはconfigファイルに入れてリークしないように管理しました。
普段以上に意識してコメントアウトを残しました。
stitchとfigmaを使ってある程度のモックを作ってから開発に取り組みました。
更新ボタンや絞り込みボタンでスクロールのtopに戻れるようにしました。

### デザイン面でこだわったポイント
片手で全ての操作が行えるようボタンのサイズは配置は調整しました。
暖色が食欲をそそる色だと調べたら分かったので、オレンジやアプリコットをメインカラーに使用しました。
シマー付きのスケルトンviewやプログレスviewを入れ、ロード中にユーザーが不安にならないようにしました。
画像を大きくして視覚的な店舗情報を強くし、店舗探しをより簡単にできるようにしました。
店舗カードと地図を丸くし全体的に柔らかい印象になるようにしました。
可能なところはSFシンボルを使用し情報過多にならないようにしました。
検索半径と表示件数は手軽に設定できるようにスライダーとピッカーで実装しました。
スクロールに応じてカードの透明度を薄くしました。
デザインの4原則は意識してテキスト等を配置しました。

### 技術面でアドバイスしてほしいポイント
- アプリ起動時にコンテントviewの描画までに時間がかかってしまうので修正したいです。
- テストはどのようにしていけばいいかがわかりませんでした。
- throwsなどでエラーが起きた時print以外にどんなことをすれば良いか知りたいです。
- 取得した画像の画質がどうしても悪く写ってしまいもし解決策があれば知りたいです。
- ダークモードの際の配色をどうすればいいかわかりませんでした。
- 今回はURLセッションを使ったのですがAlamofireとSwiftyJsonとの選定基準が知りたいです。
  
### 自己評価・感想
アプリケーションの完成度としては80点程だと考えています。
特に店舗の詳細画面をもっと凝って行きたいです。

これまで知識としてAPIキーは公開してはいけないことやWEBAPIの叩き方は知っていましたが、自力で実装まではしたことがなかったのでとてもいい経験になったし、及第点は取れたと感じています。

mapkitやshimmerは初めて使いましたが実装が想像よりは簡単で安心しました。
経路を表示できたらなお良しだったと思います。

設計は特にviewmodelが自分で全て考えることができなかったのでうまくしていかないといけないと感じました。

実装をしていく中で今まで知りもしなかった知識がついていくことを実感でき、何より自分でコードを書き実装することが楽しかったです。
